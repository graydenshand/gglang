As an analyst using multiple aesthetic channels, I want each mapped aesthetic to produce its own legend so I can read the plot without ambiguity.

# Motivating example

A bubble chart with color, size, and shape all mapped:

```gg
MAP x=:gdp, y=:life_expectancy, color=:continent, size=:population, shape=:income_group
GEOM POINT
```

This should render three legends stacked vertically in the legend region:
1. **continent** — color swatches
2. **population** — graduated circles
3. **income_group** — shape markers

# What this exercises

- **Layout**: the legend region currently assumes a single legend. Multiple legends need to be stacked within the legend area, each with its own title, items, and spacing. This likely means the legend region becomes a vertical `Split` node, or the legend rendering itself handles internal layout.
- **Merged legends**: when two aesthetics map the same variable (e.g. `color=:species, shape=:species`), ggplot2 merges them into a single legend showing both the color and shape per level. This is a nice-to-have but worth considering in the design.
- **Dynamic sizing**: the legend region width should adapt to the widest legend content (longest label + swatch). With multiple legends, total height grows — may need scrolling or overflow handling for many categories.

# Implementation notes

- `Blueprint::render()` currently produces a single `Vec<Element>` for `PlotRegion::Legend`. Needs to support N legend blocks, each generated by its respective scale.
- Each `Scale` that has a legend should expose a method like `legend_entries() -> Vec<LegendEntry>` returning labels + visual keys (color swatch, sized circle, shape marker, etc.)
- Layout: either subdivide the legend region into sub-regions per legend, or let the legend elements self-layout within a single region using relative vertical offsets.
- Legend title (the aesthetic/variable name) should appear above each legend block.

# Dependencies

- At least two legend-producing aesthetics implemented (color is done; needs shape, size, or alpha)
- More interesting with three, but testable with two
